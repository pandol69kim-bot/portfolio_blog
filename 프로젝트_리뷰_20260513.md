## 1. 프로젝트 개요

**Marshall Han**의 개인 포트폴리오 사이트. Next.js 16 + TypeScript + Tailwind CSS v4 기반의 풀스택 웹 애플리케이션.

| 항목 | 내용 |
|------|------|
| 프레임워크 | Next.js 16.2.6 (App Router) |
| 언어 | TypeScript 5 |
| 스타일링 | Tailwind CSS v4 + shadcn/ui |
| 인증 | better-auth (GitHub OAuth) |
| DB/ORM | PostgreSQL + Prisma 7 |
| 배포 대상 | Vercel 예상 |

---

## 2. 주요 기능

### 페이지 구성
| 라우트 | 설명 |
|--------|------|
| `/` | Hero 섹션 + 핵심 지표 |
| `/projects` | 프로젝트 케이스 스터디 |
| `/about` | 소개 및 역량 |
| `/experience` | 경력 타임라인 |
| `/blog` | 블로그 (CRUD) |
| `/blog/[slug]` | 블로그 상세 + 평점 |
| `/blog/new` | 새 글 작성 |
| `/blog/edit/[slug]` | 글 수정 |
| `/comments` | 방명록 (Guestbook) |
| `/contact` | 연락처 |

### 구현된 API 엔드포인트
| 엔드포인트 | 메서드 | 기능 |
|-----------|--------|------|
| `/api/auth/[...all]` | ALL | better-auth 핸들러 |
| `/api/blog` | POST | 블로그 글 생성 |
| `/api/blog/[slug]` | GET/PUT/DELETE | 글 상세 CRUD |
| `/api/blog/rate` | POST | 별점 평가 |
| `/api/comments` | GET/POST | 방명록 조회/작성 |

### 다국어 지원 (i18n)
- 지원 언어: `ko`, `en`, `ja`, `zh` (4개국어)
- 구현 방식: Context 기반 클라이언트 i18n (`lib/i18n/`)
- 전환: `LanguageToggle` 컴포넌트

## 3. 아키텍처 분석

### 디렉토리 구조
```
app/               # App Router 페이지 및 API
components/
  ui/              # shadcn/ui 기본 컴포넌트 (Button, Card, Textarea, DropdownMenu)
  portfolio/       # 포트폴리오 전용 컴포넌트 (page-intro)
  site-header.tsx  # 글로벌 헤더
  site-footer.tsx  # 글로벌 푸터
  auth-button.tsx  # 인증 버튼
  blog-form.tsx    # 블로그 폼
  star-rating.tsx  # 별점 컴포넌트
  language-toggle.tsx
  theme-toggle.tsx
  markdown-renderer.tsx
lib/
  auth.ts          # better-auth 설정
  auth-client.ts   # 클라이언트 인증 헬퍼
  prisma.ts        # Prisma 클라이언트 싱글턴
  portfolio-data.ts # 정적 포트폴리오 데이터
  utils.ts         # cn() 유틸
  i18n/            # 다국어 딕셔너리 + Context
  generated/prisma/ # Prisma 생성 클라이언트
prisma/
  schema.prisma    # DB 스키마
```

### DB 스키마 모델
- `User` — better-auth 사용자 (이메일, 이름, 이미지)
- `Session` — 세션 토큰 관리
- `Account` — OAuth 제공자 계정 (GitHub)
- `Verification` — 이메일 인증
- `BlogPost` — 블로그 글 (slug, title, content)
- `Comment` — 방명록/댓글 (User-BlogPost 연관)
- `Rating` — 별점 (User-BlogPost, unique 제약)

### 폰트 전략
- `Manrope` — 본문 산세리프 (`--font-sans`)
- `Fraunces` — 세리프 제목체 (`--font-serif`)
- `Geist Mono` — 코드 모노스페이스 (`--font-geist-mono`)

## 4. 강점

### 설계 품질
- **App Router 완전 활용**: 서버 컴포넌트 + 클라이언트 컴포넌트 분리가 의도적
- **타입 안전성**: TypeScript 전면 적용, Prisma 생성 타입 활용
- **컴포넌트 분리**: UI 레이어(shadcn)와 포트폴리오 도메인 로직 명확히 구분
- **시맨틱 HTML**: `<header>`, `<nav aria-label>`, `<main>` 등 접근성 기반 마크업
- **정적 데이터 분리**: `lib/portfolio-data.ts`로 콘텐츠와 뷰 로직 분리

### UX/디자인
- 다크/라이트 테마 (`next-themes`)
- 4개국어 런타임 전환
- sticky 헤더 + backdrop-blur
- 모바일 네비게이션 별도 처리
- `react-markdown` + `remark-gfm`으로 블로그 마크다운 렌더링

### 인증/보안
- GitHub OAuth via better-auth — 검증된 인증 라이브러리
- API Route에서 세션 기반 보호 적용
- 댓글/평점은 로그인 사용자만 가능

## 5. 개선 필요 사항

### 높음 (HIGH)

#### 1. 관리자 권한 체크 미구현
- **위치**: `app/api/blog/route.ts:16-19`
- **문제**: 블로그 글 생성/수정/삭제가 로그인한 모든 사용자에게 허용됨. 주석 처리된 admin 체크가 있으나 비활성화 상태
- **조치**: 환경변수 `ADMIN_EMAIL`과 비교하는 미들웨어 또는 유틸 함수 추가

```typescript
// 예시
if (session.user.email !== process.env.ADMIN_EMAIL) {
  return new NextResponse("Forbidden", { status: 403 });
}
```

#### 2. API 입력값 검증 없음
- **위치**: `app/api/blog/route.ts`, `app/api/comments/route.ts`
- **문제**: `req.json()` 결과를 Zod 등으로 검증하지 않고 바로 DB에 전달
- **조치**: Zod 스키마 도입하여 경계 검증 추가

#### 3. `error: any` 타입 사용
- **위치**: `app/api/blog/route.ts:35`
- **문제**: `catch (error: any)` — 규칙 위반 (`unknown` 권장)
- **조치**: `unknown`으로 변경 후 타입 가드로 `error.code` 접근

### 중간 (MEDIUM)

#### 4. 클라이언트 사이드 i18n의 SSR 한계
- **위치**: `lib/i18n/language-context.tsx`
- **문제**: Context 기반 i18n은 클라이언트에서만 동작 → 초기 SSR 시 언어 설정이 반영되지 않아 깜빡임(hydration mismatch) 가능
- **조치**: `Accept-Language` 헤더 또는 쿠키 기반 언어 감지를 미들웨어에서 처리하거나, URL 기반 i18n (`/ko/`, `/en/`) 고려

#### 5. 테스트 코드 전무
- 단위 테스트, 통합 테스트, E2E 테스트 없음
- 최소한 API Route 통합 테스트 및 컴포넌트 단위 테스트 권장

#### 6. Prisma datasource URL 누락
- **위치**: `prisma/schema.prisma:9`
- **문제**: `datasource db { provider = "postgresql" }` — `url` 필드 없음 (환경변수 설정 필요)
- **조치**: `url = env("DATABASE_URL")` 명시

### 낮음 (LOW)

#### 7. 블로그 목록 페이지네이션 미구현
- 글이 많아질 경우 성능 문제 예상

#### 8. Rate limiting 없음
- 댓글/평점 API에 요청 제한 없음

#### 9. 모바일 네비게이션 접근성
- 모바일 스크롤 네비게이션에 `scrollbar-hide` 사용 중 — 스크린리더 사용자 대상 개선 여지

## 7. 종합 평가

**전반적 완성도: 7.5/10**

포트폴리오 사이트로서 기능 구성이 충실하고 디자인 품질이 높습니다. App Router, better-auth, Prisma의 통합이 깔끔하며 다국어 지원과 테마 전환 등 UX 세부 사항에도 신경 쓴 흔적이 보입니다.

핵심 보완 과제는 **블로그 CRUD에 대한 관리자 권한 체크**와 **API 입력값 검증(Zod)**입니다. 이 두 가지만 해결하면 배포 수준의 보안 기준을 충족합니다. 테스트 코드는 프로젝트 규모에 맞게 API Route 레벨부터 단계적으로 추가하는 것을 권장합니다.
